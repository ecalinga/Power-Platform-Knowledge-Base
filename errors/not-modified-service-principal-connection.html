Low PP-0580 Dataverse Web API HTTP 304

Applies To

Power Platform connection using app registration or service principal authentication

Typical Scenario

A flow, app, or integration authenticates with an application identity instead of a personal user account.

What This Error Means

A conditional retrieval indicates the row has not changed since the supplied condition. This article focuses on the service principal connection scenario.

Common Causes

If-None-Match or modified-since condition matched the cached state. In this scenario, check app registration permissions, consent, secret expiry, certificate validity, security roles, and connection ownership.

How To Resolve It

Treat the response as a cache signal and continue using the existing data. For service principal connection, also verify: Check app registration permissions, consent, secret expiry, certificate validity, security roles, and connection ownership.

How To Prevent It

Handle 304 separately from actual failure paths. For service principal connection, document the expected permissions, connector settings, and test data used to validate the fix.
