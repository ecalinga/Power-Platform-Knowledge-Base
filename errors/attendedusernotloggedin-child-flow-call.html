AttendedUserNotLoggedIn in Child Flow Call

An attended desktop flow cannot run because the user is not logged in. This article focuses on the child flow call scenario.

Medium PP-0693 Power Automate Desktop Desktop flow queue

Applies To

Power Automate parent flow calling a child flow

Typical Scenario

A parent flow invokes reusable child-flow logic and waits for a result.

What This Error Means

An attended desktop flow cannot run because the user is not logged in. This article focuses on the child flow call scenario.

Common Causes

No active logged-in Windows session exists for the attended automation user. In this scenario, check solution membership, trigger type, response action, input schema, connection references, and child-flow ownership.

How To Resolve It

Log in as the target user and confirm the Power Automate service account has the needed machine permissions. For child flow call, also verify: Check solution membership, trigger type, response action, input schema, connection references, and child-flow ownership.

How To Prevent It

Schedule attended flows only when the user session is reliably available. For child flow call, document the expected permissions, connector settings, and test data used to validate the fix.
