High PP-0751 Power Platform Gateway HTTP 502

Applies To

Power Automate or Power Apps using SQL Server through a connector or gateway

Typical Scenario

The app or flow queries, inserts, updates, or deletes SQL Server records as part of a business process.

What This Error Means

The on-premises data gateway or downstream service is unavailable when the platform tries to reach it. This article focuses on the sql server connector scenario.

Common Causes

Gateway offline, network path unavailable, service stopped, firewall rule changed, or backend data source unavailable. In this scenario, check SQL permissions, gateway mapping, stored procedure parameters, table schema changes, and query timeout behavior.

How To Resolve It

Confirm gateway status, restart gateway services if needed, test network reachability, and validate backend data-source availability. For sql server connector, also verify: Check SQL permissions, gateway mapping, stored procedure parameters, table schema changes, and query timeout behavior.

How To Prevent It

Monitor gateway uptime and alert on offline gateway clusters. For sql server connector, document the expected permissions, connector settings, and test data used to validate the fix.
