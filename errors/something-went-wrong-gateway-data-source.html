Something went wrong [5objp] in On-premises Gateway Data Source

A wrapped Power Apps mobile app fails during authentication even though the build may have succeeded. This article focuses on the on-premises gateway data source scenario.

High PP-0710 Power Apps Wrap

Applies To

Power Automate, Power Apps, or connector using an on-premises data gateway

Typical Scenario

The automation or app reaches SQL Server, file shares, or internal systems through a gateway connection.

What This Error Means

A wrapped Power Apps mobile app fails during authentication even though the build may have succeeded. This article focuses on the on-premises gateway data source scenario.

Common Causes

APK signature hash mismatch or redirect URI mismatch in Microsoft Entra app registration. In this scenario, check gateway online status, data source mapping, credentials, network reachability, and connector permissions.

How To Resolve It

Generate the correct hash from the signing keystore, update Android authentication settings, and verify redirect URI casing and encoding. For on-premises gateway data source, also verify: Check gateway online status, data source mapping, credentials, network reachability, and connector permissions.

How To Prevent It

Control keystore, hash, bundle ID, and redirect URI values in release documentation. For on-premises gateway data source, document the expected permissions, connector settings, and test data used to validate the fix.
