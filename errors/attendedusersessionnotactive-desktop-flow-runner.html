AttendedUserSessionNotActive in Desktop Flow Runner

An attended desktop flow cannot run because the target user session is not active. This article focuses on the desktop flow runner scenario.

Medium PP-0668 Power Automate Desktop Desktop flow queue

Applies To

Power Automate Desktop attended or unattended run

Typical Scenario

A desktop flow starts on a machine or machine group and attempts to create or reuse a Windows session.

What This Error Means

An attended desktop flow cannot run because the target user session is not active. This article focuses on the desktop flow runner scenario.

Common Causes

User is signed out, locked, disconnected, or not available in the expected interactive session. In this scenario, check machine availability, credentials, RDP policy, user session state, gateway registration, and installed credential providers.

How To Resolve It

Have the attended user sign in and keep the session active, or redesign the process as unattended. For desktop flow runner, also verify: Check machine availability, credentials, RDP policy, user session state, gateway registration, and installed credential providers.

How To Prevent It

Use unattended flows for server-like automation and attended flows only for user-present tasks. For desktop flow runner, document the expected permissions, connector settings, and test data used to validate the fix.
