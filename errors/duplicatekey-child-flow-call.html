DuplicateKey in Child Flow Call

A Dataverse write conflicts with an existing alternate key value. This article focuses on the child flow call scenario.

Medium PP-1173 Dataverse Data validation HTTP 409

Applies To

Power Automate parent flow calling a child flow

Typical Scenario

A parent flow invokes reusable child-flow logic and waits for a result.

What This Error Means

A Dataverse write conflicts with an existing alternate key value. This article focuses on the child flow call scenario.

Common Causes

Attempted insert with a key already used by another row or concurrent create using the same key. In this scenario, check solution membership, trigger type, response action, input schema, connection references, and child-flow ownership.

How To Resolve It

Find the row with the existing key, decide whether to update or skip, and retry only if the key is unique. For child flow call, also verify: Check solution membership, trigger type, response action, input schema, connection references, and child-flow ownership.

How To Prevent It

Implement idempotent upsert patterns for integration loads. For child flow call, document the expected permissions, connector settings, and test data used to validate the fix.
