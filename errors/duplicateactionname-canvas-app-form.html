DuplicateActionName in Canvas App Form

Two actions use the same internal action name inside the flow definition. This article focuses on the canvas app form scenario.

Medium PP-0045 Power Automate Cloud flow design-time

Applies To

Power Apps canvas app form or formula

Typical Scenario

The user saves a form, loads a record, calls a flow, or uses connector data inside a canvas app.

What This Error Means

Two actions use the same internal action name inside the flow definition. This article focuses on the canvas app form scenario.

Common Causes

Copied actions, manually edited JSON, or renamed steps that collide with an existing action name. In this scenario, check app sharing, data source permissions, formula null handling, and connector policy restrictions.

How To Resolve It

Rename one duplicate action and update expressions that reference the old action name. For canvas app form, also verify: Check app sharing, data source permissions, formula null handling, and connector policy restrictions.

How To Prevent It

Rename copied actions immediately and avoid manual edits that duplicate action keys. For canvas app form, document the expected permissions, connector settings, and test data used to validate the fix.
