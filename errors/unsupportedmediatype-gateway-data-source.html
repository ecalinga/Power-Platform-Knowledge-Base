UnsupportedMediaType in On-premises Gateway Data Source

The service rejected the payload content type. This article focuses on the on-premises gateway data source scenario.

Medium PP-0990 Power Platform Connector/API HTTP 415

Applies To

Power Automate, Power Apps, or connector using an on-premises data gateway

Typical Scenario

The automation or app reaches SQL Server, file shares, or internal systems through a gateway connection.

What This Error Means

The service rejected the payload content type. This article focuses on the on-premises gateway data source scenario.

Common Causes

Missing or wrong Content-Type header, binary sent where JSON is expected, or unsupported file format. In this scenario, check gateway online status, data source mapping, credentials, network reachability, and connector permissions.

How To Resolve It

Set the expected Content-Type, convert the payload to the supported format, and retry. For on-premises gateway data source, also verify: Check gateway online status, data source mapping, credentials, network reachability, and connector permissions.

How To Prevent It

Validate headers in custom connectors and HTTP actions. For on-premises gateway data source, document the expected permissions, connector settings, and test data used to validate the fix.
